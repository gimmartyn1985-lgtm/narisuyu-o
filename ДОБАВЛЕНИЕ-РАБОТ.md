# Как добавлять новые работы

Теперь у сайта есть простая админка.

Адрес после деплоя:

```text
https://narisuyu-o.vercel.app/admin
```

## Что нужно один раз подготовить

Нужен GitHub fine-grained token с доступом к репозиторию `gimmartyn1985-lgtm/narisuyu-o`.

Минимальные права:

- `Contents`: `Read and write`

Лучше создавать токен в том GitHub-аккаунте, который владеет репозиторием и связан с Vercel. Тогда публикации обычно доходят до автодеплоя без лишних сюрпризов.

## Как добавить работу

1. Откройте страницу `/admin`.
2. Введите:
   - репозиторий `gimmartyn1985-lgtm/narisuyu-o`
   - ветку `main`
   - GitHub token
3. Нажмите `Сохранить доступ`.
4. Выберите изображение работы.
5. Заполните:
   - название
   - подпись
   - описание изображения
6. Нажмите `Опубликовать работу`.

После этого админка:

- загрузит изображение в папку `фото/`
- обновит файл `works.json`
- создаст коммит в GitHub

Через короткое время Vercel подтянет новую версию сайта.

## Важные заметки

- Токен сохраняется только в браузере на этом устройстве.
- В код сайта токен не записывается.
- Если устройство меняется, токен нужно ввести заново.
- Если токен больше не нужен, его лучше отозвать в GitHub.

## Ручной запасной вариант

Если админка временно недоступна, каталог можно обновить вручную через репозиторий:

1. Загрузить изображение в папку `фото/`
2. Добавить новую запись в `works.json`

Формат записи:

```json
{
  "title": "Название работы",
  "meta": "Краткое описание техники",
  "image": "фото/имя-файла.jpg",
  "alt": "Краткое описание рисунка для сайта"
}
```
